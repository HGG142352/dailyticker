표	계산 공식 (DART account_nm 기준)
EPS	(account_nm: "당기순이익") / (발행주식수)
PER	(현재 주가) / (계산된 EPS)
BPS	(account_nm: "자본총계") / (발행주식수)
PBR	(현재 주가) / (계산된 BPS)
ROE	(account_nm: "당기순이익") / (account_nm: "자본총계") * 100
ROA	(account_nm: "당기순이익") / (account_nm: "자산총계") * 100 발행주식수 확보: DART의 fnltt API에는 주식수가 없습니다. 공시 리스트(list.json)에서 '주식의 총수' 항목을 파싱하거나, 네이버 금융/야후 파이낸스 같은 외부 API에서 total_shares를 가져와야 합니다.

계정명 일관성: 대부분 account_nm: "당기순이익"을 쓰지만, 연결재무제표일 경우 "당기순이익(비지배지분포함)" 또는 "지배기업소유주지분 당기순이익" 중 어떤 것을 쓸지 결정해야 합니다. (퀀트에서는 보통 지배주주순이익을 선호합니다.)

단위 주의: API 응답값인 thstrm_amount는 '원' 단위입니다. 계산 시 억 단위나 만 단위로 치환하여 가독성을 높이는 것이 좋습니다.
